## Key positive findings

### 1) Type safety được cải thiện rõ rệt

- Nhiều `callRpc<any>` được thay bằng:
  - `callRpc<Equipment[]>`
  - `callRpc<Equipment | null>`
  - `callRpc<boolean>`
  - `callRpc<unknown>` rồi map qua hàm narrowing
- Nhiều `catch (e: any)`/`onError(error: any)` đổi sang `unknown` + guard.

### 2) Tách mapper/types hợp lý, giảm phình file

- `use-cached-maintenance.ts` được rút gọn bằng cách tách:
  - `use-cached-maintenance.types.ts`
  - `use-cached-maintenance.rpc.ts`
- Inventory/report logic mapping chuyển sang `use-inventory-data.types.ts`, giúp hook chính sạch hơn.

### 3) Fix được bug logic có thật (inventory export)

- Luồng `thanh_ly` trước đây có nguy cơ double-count (vừa vào transfer path vừa vào liquidation path).
- Cấu trúc mới trong `mapExportedInventoryItems` xử lý theo nhánh rõ hơn, tránh duplicate cho case hoàn thành.

### 4) Backward compatibility giữ ổn

- Các type/interface public được re-export lại từ file types mới.
- Không thấy thay đổi wire contract RPC.

### 5) Test coverage được bổ sung đúng trọng tâm

## Important technical notes by area

### A. `src/hooks/use-cached-equipment.ts`

- Bỏ cast `user as any` bằng helper `toCacheScopeUser`.
- Mutation `equipment_update` xử lý return boolean đúng hơn:
