# Device Quota Knowledge Base (Updated January 2025)

This document consolidates knowledge from NotebookLM research about Vietnamese medical equipment quota regulations, superseding the outdated plan based on Circular 08/2019/TT-BYT.

## 1. Legal Framework Update

### Current Regulations (As of 2025-2026)

| Regulation | Status | Effective Date | Purpose |
|------------|--------|----------------|---------|
| **Circular 46/2025/TT-BYT** | **PRIMARY** | Feb 15, 2026 | Decentralizes quota authority to facility heads |
| **Circular 08/2019/TT-BYT** | Still Valid (Criteria) | Aug 1, 2019 | Defines methodology for quota calculation |
| **Circular 01/2026/TT-BYT** | New | Mar 1, 2026 | National centralized procurement for specific items |

### Key Changes in Circular 46/2025/TT-BYT
1. **Decentralized Authority**: Hospital Directors (not Ministry) now approve quotas
2. **Self-Governance**: Facilities create/adjust their own quota decisions
3. **Accountability**: Directors are fully responsible before the Minister of Health
4. **Public Transparency**: Mandatory publication on Ministry's Electronic Portal

### Relationship Between Circulars
- **Circular 46/2025** changes WHO approves quotas (facility head vs. Ministry)
- **Circular 08/2019** still defines HOW to calculate quotas (criteria, methodology)
- Transitional: Existing quotas valid until replaced by new decisions under 46/2025

## 4. Revised Database Schema

### Table: quyet_dinh_dinh_muc (Quota Decisions)
```sql
CREATE TABLE public.quyet_dinh_dinh_muc (
  id bigint generated by default as identity not null primary key,
  don_vi_id bigint references don_vi(id) not null,
  so_quyet_dinh text not null,           -- e.g., "15/QĐ-BVBM"
  ngay_ban_hanh date not null,           -- Date signed
  nguoi_ky text,                         -- Name of Director
  chuc_vu_nguoi_ky text,                 -- Position (Giám đốc)
  hieu_luc_tu date not null,             -- Valid from
  hieu_luc_den date,                     -- Valid until (null = indefinite)
  trang_thai text default 'draft',       -- 'draft', 'active', 'replaced'
  file_dinh_kem text,                    -- URL to scanned PDF
  da_cong_khai boolean default false,    -- Published to Ministry portal?
  ngay_cong_khai timestamp,              -- When published
  ghi_chu text,
  created_by bigint,
  updated_by bigint,
  created_at timestamp with time zone default now(),
  updated_at timestamp with time zone default now(),

  -- Version control
  phien_ban int default 1,
  thay_the_cho_id bigint references quyet_dinh_dinh_muc(id)
);
```

### Table: nhom_thiet_bi_dinh_muc (Equipment Category Hierarchy)
```sql
CREATE TABLE public.nhom_thiet_bi_dinh_muc (
  id bigint generated by default as identity not null primary key,
  parent_id bigint references nhom_thiet_bi_dinh_muc(id),
  ma_nhom text not null,                 -- e.g., "I", "A", "1", "a"
  ten_nhom text not null,                -- e.g., "Chẩn đoán hình ảnh"
  ten_nhom_en text,                      -- English name
  loai_cap text not null,                -- 'cap_nhom', 'cap_hang_muc', 'cap_thiet_bi'
  phan_loai text,                        -- 'A' (đặc thù) or 'B' (khác)
  thu_tu int default 0,                  -- Display order
  is_leaf boolean default false,         -- True for equipment items
  don_vi_tinh text,                      -- Only for leaf nodes
  ghi_chu text,
  created_at timestamp with time zone default now()
);
```

### Table: chi_tiet_dinh_muc (Quota Details)
```sql
CREATE TABLE public.chi_tiet_dinh_muc (
  id bigint generated by default as identity not null primary key,
  quyet_dinh_id bigint references quyet_dinh_dinh_muc(id) not null,
  nhom_thiet_bi_id bigint references nhom_thiet_bi_dinh_muc(id) not null,
  don_vi_tinh text not null,             -- e.g., "Hệ thống", "Cái"
  so_luong_dinh_muc int not null,        -- Maximum allowed quantity
  so_luong_toi_thieu int,                -- Minimum required (optional)
  khoa_phong_ap_dung text,               -- Specific department (optional)
  can_cu_tinh_toan text,                 -- Calculation basis per Circular 08/2019
  ghi_chu text,

  -- National centralized procurement flag
  mua_sam_tap_trung boolean default false,

  created_at timestamp with time zone default now()
);
```

### Table: lich_su_dinh_muc (Quota Change History)
```sql
CREATE TABLE public.lich_su_dinh_muc (
  id bigint generated by default as identity not null primary key,
  chi_tiet_id bigint references chi_tiet_dinh_muc(id),
  quyet_dinh_id bigint references quyet_dinh_dinh_muc(id),
  thao_tac text not null,                -- 'tao', 'cap_nhat', 'dieu_chinh', 'huy'
  snapshot_truoc jsonb,
  snapshot_sau jsonb,
  ly_do text,
  thuc_hien_boi bigint,
  thoi_diem timestamp with time zone default now()
);
```

## 5. Workflow Changes

### Quota Approval (Simplified - Per Circular 46/2025)

```
1. Department Request
       ↓
2. Internal Review (Facility Council)
       ↓
3. Director Approval & Signing
       ↓
4. Mandatory Public Disclosure (Ministry Portal)
       ↓
5. Effective & Enforced
```

**Key Difference**: No Ministry approval step in workflow. Director is final approver.

### Over-Quota Handling

1. **Identify Need**: Facility determines current quota insufficient
2. **Justify**: Must align with Circular 08/2019 criteria:
   - Approved functions and tasks
   - Actual bed count and service frequency
   - Infrastructure and workforce capacity
3. **Adjust**: Director issues new/amended Quota Decision
4. **Publish**: Post to Ministry portal
5. **Procure**: Only then can procurement proceed

## 6. National Centralized Procurement (Circular 01/2026)

### Affected Items (Starting Jan 1, 2027)

| Category | Items | Procurement Method |
|----------|-------|-------------------|
| **Thủy tinh thể nhân tạo** | Acrylic IOLs, hydrophobic/hydrophilic, aspheric | National bidding only |
| **Giá đỡ động mạch vành** | Drug-eluting stents (sirolimus/everolimus), polymer-coated | National bidding only |

### System Requirements
- Flag these items in quota management
- Disable local bidding workflows for flagged items
- Integrate demand registration with National Procurement Center

## 7. Reporting & Audit Requirements

### Mandatory Reports
1. **Public Disclosure Report**: Quota Decision must be published on Ministry portal
2. **Compliance Report**: Inventory vs. approved quota comparison
3. **Price Listing**: Equipment prices with configuration and tax info

### Audit Points
- Current inventory matches approved quota decision
- Quota decision published on Ministry portal
- Any excess has corresponding adjustment decision signed BEFORE procurement
- Centralized procurement items bought through national channel
